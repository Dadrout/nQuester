Детальный Гейм-дизайн Документ: “nQuester: Incubator Rush”

Версия 2.0 (Детализированная)

1. Высокоуровневая Концепция

  • Название игры: nQuester: Incubator Rush
  • Жанр: 2D Top-Down RPG с квестовой прогрессией. (Изменено с платформера для ускорения разработки и фокуса на контенте).
  • Движок: Pygame
  • Целевая аудитория: Участники и фанаты хакатона nFactorial.
  • Логлайн: Будучи новым студентом в хаотичном инкубаторе nFactorial, вы должны привлечь 10,000 пользователей для своего стартапа до Demo Day, выполняя уморительные и сложные квесты для реальных менторов, мемифицированных студентов и других причудливых персонажей.

2. Ключевой Игровой Цикл

Игровой процесс представляет собой цикл, состоящий из следующих состояний (game states):

  1. Состояние Исследования (Exploration State):
  • Игрок свободно перемещается по карте с видом сверху.
  • Основная цель: находить NPC с маркером квеста (!).
  • Возможно взаимодействие с интерактивными объектами окружения.
  1. Состояние Диалога (Dialogue State):
  • Активируется при взаимодействии с NPC.
  • Игровой мир "замирает", на экране появляется диалоговое окно.
  • Игрок читает текст и может делать выбор в диалоге, что приводит к принятию квеста или получению информации.
  1. Состояние Мини-игры (Minigame State):
  • Активируется как часть квеста.
  • Экран переключается на интерфейс мини-игры (например, викторина, пазл).
  • Управление и цели меняются в соответствии с правилами мини-игры.
  • После завершения игра возвращается в состояние исследования.
  1. Состояние Выполнения Квеста (Questing State):
  • Игрок активно выполняет цель квеста (например, ищет предмет).
  • В интерфейсе отображается текущая задача.
  • После выполнения цели игрок возвращается к NPC для сдачи квеста, переходя в состояние диалога.

3. Персонаж Игрока и Управление

  • Персонаж: Алекс, решительный, но немного перегруженный студент nFactorial.
  • Управление (Top-Down):
  • W, A, S, D / Стрелки: Движение вверх, влево, вниз, вправо.
  • E: Взаимодействие (Начать диалог, поднять предмет, использовать терминал).
  • Q / Tab: Открыть журнал квестов (UI).
  • Параметры (для player.py):
  • speed = 3 (пикселей за кадр)
  • animation_speed = 0.1 (скорость смены кадров анимации)
  • quest_log = [] (список ID активных квестов)
  • inventory = {} (словарь с квестовыми предметами)

4. Игровой Мир и Дизайн Уровней

  • Сеттинг: Единый, большой, взаимосвязанный 2D-уровень "Кампус nFactorial" с видом сверху. Карта построена на тайлах (например, 32x32 пикселя).
  • Зоны:
  1. The Commons (Общая зона): Центральный хаб. Здесь много открытого пространства, столы для работы, диваны. Большинство студентов-NPC находятся здесь. Ключевые проходы ведут в другие зоны.
  2. The Mentor's Den (Логово Менторов): Тихая зона с отдельными "кабинетами" (небольшие комнаты 3x3 тайла). Каждый ментор сидит за своим столом. Здесь царит деловая атмосфера.
  3. The Vibe-Coding Zone (Зона Вайб-кодинга): Темная комната с неоновой подсветкой, пуфиками и постерами на стенах. Здесь находится "Wabcoder 9000" и звучит lofi-музыка.
  4. The Kitchen (Кухня): Хаотичное место. Разбросаны чашки, коробки из-под пиццы. Здесь можно найти квестовые предметы вроде "Энергетик" и "Последний кусочек пиццы".

5. Система NPC и Квестов

Основная цель — привлечь 10,000 пользователей.

Детализированные Квесты

  1. Алихан Губаев (iOS Mentor)
  • Квест: “Swift Debug Showdown”
  • Геймплей: При взаимодействии открывается UI-окно с 5 строками кода. Одна из них содержит очевидную ошибку (например, let name = "Alex" вместо var name = "Alex"). Игрок должен кликнуть на неверную строку. Таймер на 15 секунд.
  • Награда: +500 Пользователей, -3 Бага.
  1. Алибек Сеитов (AI/ML Mentor)
  • Квест: “Train Your Brain”
  • Геймплей: Открывается мини-игра. В центре экрана "сырая" модель (иконка мозга). Вокруг появляются иконки "Данные с котиками" и "Данные с собаками". Игрок должен перетащить 5 иконок с котиками к модели, игнорируя собак.
  • Награда: +800 Пользователей, +1 AI Insight.
  1. Бахредин Зургамбаев (TypeScript Mentor)
  • Квест: “Strict Mode Madness”
  • Геймплей: Викторина. На экране появляются 3 вопроса подряд. Пример: const user = { id: 1, name: 'Alex' }; Каков тип user.id? Варианты: A) string, B) number, C) any. Игрок должен выбрать правильный ответ.
  • Награда: +600 Пользователей, +1 Type Safety Certificate.
  1. Wabcoder 9000 (Meme NPC)
  • Квест: “AI Code Wizard”
  • Геймплей: Игрок должен найти в другой части кампуса предмет "Древний манускрипт по промптам". После этого вернуться к Wabcoder 9000. При взаимодействии открывается терминал, куда нужно ввести точный текст с манускрипта: solve my problems with vibecoding.
  • Награда: +1500 Пользователей.
  1. Bug Slime (Enemy NPC)
  • Квест: “Bug Bash”
  • Геймплей: Вместо прыжка, игрок должен найти предмет "Отладчик" (выглядит как молоток). Взяв его, игрок может подойти к Bug Slime и нажать "E", чтобы "ударить" его. Нужно 3 удара. Bug Slime после каждого удара телепортируется в другую часть комнаты.
  • Награда: +300 Пользователей, -1 Баг.

6. Интерфейс (UI / HUD)

  • Top-Left: Пользователи: [current_users] / 10,000 (с иконкой человечка).
  • Top-Right: Текущий квест: [quest_title] (появляется, когда квест активен).
  • Bottom-Center: Диалоговое окно. Должно вмещать 3-4 строки текста, имя NPC и его аватар.
  • Инвентарь/Журнал (по кнопке Q): Простое окно со списком названий активных квестов и иконками квестовых предметов.

7. Условия Победы и Поражения

  • Победа: Достигнуто 10,000 пользователей. Запускается финальная кат-сцена "Demo Day".
  • Концовки:
  1. Launch God (выполнены все квесты менторов): "Ваш проект взорвал Product Hunt! Инвесторы выстроились в очередь."
  2. Niche Hero (10,000 пользователей, но не все квесты выполнены): "Не идеально, но ваши пользователи в восторге. Вы создали настоящее комьюнити."
  3. Burnout Legend (если будет таймер и он истечет): "Вы не запустились, но выжили. Вот вам стикер. А теперь идите спать."

8. Детальный Технический План (Pygame)

Структура Файлов и Логика

  • main.py: Главный цикл игры. Управляет состояниями (меню, игра, пауза). Создает экземпляры Player, Level, QuestManager. В цикле вызывает events(), update(), draw().
  • settings.py: Константы: SCREEN_WIDTH, SCREEN_HEIGHT, FPS, TILE_SIZE, цвета, пути к ассетам.
  • player.py: Класс Player.
  • __init__: Загружает спрайты, устанавливает rect, position (вектор), speed.
  • get_input(): Обрабатывает нажатия клавиш для движения и взаимодействия.
  • update(): Вызывает get_input(), обновляет анимацию в зависимости от вектора движения.
  • npc.py: Базовый класс NPC и подклассы.
  • NPC: __init__ с name, dialogue, quest_id.
  • MentorNPC, StudentNPC: могут иметь свою логику или просто разные спрайты.
  • quest_manager.py: Класс QuestManager.
  • __init__: Загружает quests.json в словарь self.quests_data.
  • start_quest(quest_id): Добавляет квест в player.quest_log и помечает как активный.
  • check_completion(quest_id, player): Проверяет, выполнены ли условия квеста (например, есть ли нужный предмет в player.inventory).
  • complete_quest(quest_id): Удаляет квест из активных, выдает награду.
  • level.py: Класс Level.
  • __init__: Загружает карту уровня (из TMX файла или простого текстового файла), создает группы спрайтов для стен (obstacles) и NPC.
  • run(): Вызывает update() и draw() для всех спрайтов.
  • ui.py: Класс UI.
  • display_hud(player): Отрисовывает счетчик пользователей.
  • display_dialogue(npc_name, text): Отрисовывает диалоговое окно.
  • minigames.py: Набор функций или классов для каждой мини-игры. Например, run_quiz_minigame(quest_data) будет содержать свой собственный игровой цикл, который возвращает True при победе и False при поражении.
  • data/quests.json: Расширенная структура.

9. Список Ассетов (Детализированный)

  • Спрайты (в стиле pixel art, 32x32 или 64x64 пикселя):
  • Игрок (Алекс): Анимации ходьбы (вверх, вниз, влево, вправо), анимация бездействия.
  • NPC: По одному уникальному спрайту для каждого из 8 менторов. 3-4 варианта спрайтов для студентов. Спрайт для Wabcoder 9000, Bug Slime.
  • Предметы: Иконки для water_bottle, energy_drink, debugger_hammer, prompt_manuscript.
  • Тайлсет (32x32 пикселя):
  • Набор тайлов для пола (разные для каждой зоны), стен, столов, компьютеров, диванов, растений.
  • UI:
  • Рамка для диалогового окна (9-patch).
  • Шрифт в стиле pixel art.
  • Иконки квестов (!, ?).
  • Звуки:
  • SFX: Шаги, начало диалога (бип), выполнение квеста (фанфары), получение предмета.
  • Музыка: Отдельный трек для каждой зоны (для The Vibe-Coding Zone — lofi hip-hop).
